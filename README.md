# \## 📖 О проекте

# 

# \*\*LUTOhayse.Bots\*\* — это полноценная ERP-система для мебельной фабрики, упакованная в чат-боты. Проект решает три ключевые задачи:

# 

# 1\. \*\*Клиентский интерфейс\*\* (Telegram + VK) — позволяет пользователям конструировать диваны из модулей, выбирать материалы, оформлять заказы и отслеживать их статус.

# 2\. \*\*Менеджерский бот\*\* (Telegram) — даёт администраторам и работникам фабрики управлять заказами, назначать исполнителей на каждую позицию, менять статусы и вести базу сотрудников.

# 3\. \*\*Общая файловая база\*\* (Excel) — все три бота работают с одними и теми же `.xlsx`-файлами, что делает систему \*\*портативной\*\* и \*\*не требующей установки SQL\*\*.

# 

# > 🔥 \*\*Более 50% пользователей\*\* собирают индивидуальный диван через конструктор, а не выбирают из готового каталога.  

# > 📈 \*\*Конверсия в покупку\*\* выросла в 4 раза после внедрения ботов.  

# > ⚡ \*\*Скорость обработки заказов\*\* увеличилась на 22% благодаря автоматическому распределению задач между швеями, столярами, обтяжчиками и упаковщиками.

# 

# \---

# 

# \## 🧠 Технические особенности

# 

# | Особенность | Реализация |

# |-------------|-------------|

# | \*\*Без SQL\*\* | Все данные хранятся в Excel-файлах (OpenPyXL). Система сама создаёт таблицы при первом запуске. |

# | \*\*Кроссплатформенность\*\* | Клиент доступен одновременно в Telegram и VK с единой бизнес-логикой. |

# | \*\*Конструктор диванов\*\* | Модульная сборка: базовый модуль + расширения. Схема дивана (cells.png) генерируется заранее и подгружается из папки `Photo/Construction/`. |

# | \*\*Управление ролями\*\* | Администраторы, менеджеры, швеи, столяры, обтяжчики, упаковщики. Каждый видит только свои задачи. |

# | \*\*Аутентификация\*\* | Доступ к менеджерскому боту защищён паролем (хранится в `password.txt`). Возможность отключения. |

# | \*\*Пагинация\*\* | Списки заказов (10 на страницу) и работников пролистываются через inline-клавиатуры. |

# | \*\*Медиа\*\* | Автоматическая отправка групп фотографий (InputMediaPhoto) для товаров, материалов и схем диванов. |

# | \*\*Генерация кодов\*\* | Уникальный код заказа (10 символов) и номер задачи (7 цифр) создаются случайно. |

# \## 📁 Структура репозитория

# LUTOhayse.bots/

# ├── Script\_LutoManagerBot.py      # Менеджерский бот (Telegram)

# ├── Script\_TG\_LutoClientBot.py    # Клиентский бот (Telegram)

# ├── Script\_VK\_LutoClientBot.py    # Клиентский бот (VK)

# ├── DataBase/                     # Автоматически создаваемые Excel-файлы

# │   ├── Orders\_log.xlsx

# │   ├── Products\_log.xlsx

# │   ├── Materials\_log.xlsx

# │   ├── Moduls\_log.xlsx

# │   ├── Sofas\_log.xlsx

# │   ├── Cart\_log.xlsx

# │   ├── password.txt

# │   └── id\_workers.txt

# └── Photo/                        # Фотографии и схемы

# &#x20;   ├── Products/

# &#x20;   ├── Materials/

&#x20;   └── Construction/             # Папки CXXXX с cells.png

##⚙️ Функции скриптов
===



# 1\. Script\_TG\_LutoClientBot.py — клиентский бот Telegram



* # start() — приветствие, главное меню.

# 

* # handle\_sofa\_construction() — конструирование дивана: выбор типа → базовый модуль → добавление/удаление модулей → показ схемы и цен.

# 

* # handle\_material\_selection() — выбор материала по коду или через каталог (с пагинацией по цветам).



* # save\_to\_cart() / get\_user\_cart() — корзина с возможностью изменения материала и удаления.

# 

* # save\_order() — запись заказа в Orders\_log.xlsx (каждая позиция — отдельная строка). Отправка уведомления менеджеру.

# 

* # show\_user\_orders() — вывод заказов пользователя с inline-кнопкой «Подробнее» (позиции, материалы, статус).

# 

* # get\_available\_materials\_for\_sofa() — динамический расчёт доступности материала для всех модулей дивана (цена > 0).

# 

* # send\_media\_group() — загрузка и отправка фотографий из Photo/Products/ и Photo/Materials/.

# 

# 2\. Script\_VK\_LutoClientBot.py — клиентский бот VK



* # Использует VkBotLongPoll для приёма сообщений и callback-кнопки для интерактива.

# 

* # handle\_sofa\_construction() — полностью повторяет логику Telegram-бота, но с адаптацией под VK Keyboard (VkKeyboard, VkKeyboardColor).

# 

* # upload\_photos() — загрузка фото в сообщения VK через VkUpload.photo\_messages.

# 

* # create\_cart\_table() — инициализация корзины с колонкой «Мессенджер» (VK/Telegram).

# 

* # save\_order\_from\_cart() — аналогично Telegram, но с формированием ссылки vk.com/id{user\_id}.

# 

* # handle\_order\_collection() — пошаговый сбор ФИО, телефона, адреса, комментария с возможностью пропуска.

# 

* # handle\_callback() — обработка inline-действий: просмотр деталей заказа, смена материала в корзине, удаление позиции.

# 

# 3\. Script\_LutoManagerBot.py — менеджерский бот Telegram

# Аутентификация: глобальный хук на все сообщения от неавторизованных пользователей (@bot.message\_handler(func=lambda message: not is\_authenticated(...))).

# 

* # load\_auth\_data() — чтение id\_workers.txt и password.txt с автоматическим определением кодировки (UTF-8, CP1251, Latin-1).

# 

* # get\_orders\_by\_status() — выборка заказов из Excel с группировкой по коду заказа и сбором всех позиций.

# 

* # show\_orders\_page() — пагинация заказов (10 на страницу). Каждый заказ выводится с информацией о клиенте и inline-кнопками «Изменить статус», «Удалить заказ», «Написать».

# 

* # edit\_item\_handler() — редактирование конкретной позиции: назначение швеи, столяра, обтяжчика, упаковщика через выбор из списка работников (сортировка: специалист → без роли → администратор) или ручной ввод ФИО.

# 

* # update\_task\_readiness() — обновление готовности задачи (швея/столяр и т.д.) из раздела «Мои задачи».

# 

* # update\_fio\_in\_orders() — при изменении ФИО работника автоматически обновляются все заказы, где он был назначен.

# 

* # delete\_previous\_messages() — «умная» очистка чата: удаление сообщений между двумя якорями, чтобы не засорять историю.

# 

* # main\_keyboard() — динамическая клавиатура: для администраторов показываются кнопки «Заказы» и «Работники», для остальных — только «Мой профиль» и «Мои задачи».

# 

# \##🏆 Ключевые достижения проекта





# ✅ Мультиканальность — один бизнес-процесс работает одновременно в Telegram и VK.

# 

# ✅ Отсутствие SQL-сервера — вся логика на Excel, что упрощает развёртывание и резервное копирование (достаточно скопировать папку DataBase).

# 

# ✅ Гибкая система ролей — каждый сотрудник видит только свои задачи, администратор управляет всем.

# 

# ✅ Конструктор диванов — пользователь собирает диван пошагово, видит схему и актуальные цены для каждого типа материала.

# 

# ✅ Автоматическое уведомление менеджера — при новом заказе в Telegram приходит оповещение.

# 

✅ Поддержка фотографий — товары, материалы и схемы диванов загружаются группами, порядок сортируется по имени файла (1.jpg, 2.jpg…).
